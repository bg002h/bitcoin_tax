## Critical

**[CRITICAL] crates/btctax-core/src/tax/tables.rs:175 — `se_addl_medicare_threshold` gives QSS the $250,000 joint threshold; the law and the 2024 Form 8959 put QSS at $200,000.**
§3101(b)(2)/§1401(b)(2) grant $250,000 only "in the case of a joint return"; a §2(a) surviving spouse does not file a joint return, so QSS falls under "(C) in any other case — $200,000". The 2024 Form 8959 threshold chart and the Schedule 2 L11 instructions (i1040gi--2024 p.99: "$200,000 if single, head of household, **or qualifying surviving spouse**") say so explicitly — as does this project's own recon, `design/full-return/recon/deep/02-…md:203` ("Single·HoH·QSS 200,000"). Failing scenario: QSS filer, Σ box5 = $240,000 → correct 8959 Part I = 0.9% × $40,000 = **$360.00** → Sch 2 L11; the code (`other_taxes.rs:58` → `form_8959`) computes $0 — the filed absolute return **silently understates total tax by $360** (and overstates the refund by the same, since Part V still credits the employer-withheld $360 on 25c). The same helper feeds the frozen `se.rs:143` Part-II clamp, so a QSS SE filer in the $200k–$250k window is understated on both Part I and Part II; that half predates P4 (introduced at `87e443d`), but P4 is what put it on the filed return. Fix in unfrozen `tables.rs` (move `Qss` to the $200,000 arm + KAT); **do not** touch `niit_threshold` — §1411(b)(1) *explicitly includes* "a surviving spouse" at $250,000, so the current asymmetry-free grouping there is correct (the recon's two charts at deep/02:171 vs :203 capture exactly this asymmetry). No frozen file changes are needed and no existing test pins the wrong QSS value.
